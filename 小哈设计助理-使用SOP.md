# 小哈设计助理 · 使用 SOP

> 面向哈啰两轮设计中心同事的上手手册
> 产品定位：视觉设计师的桌面 AI 工作助手（桌宠形态，IP 交互入口）
> 版本：**v2.1（功能试行阶段）**｜适用：macOS｜更新：2026-07

## 一、这是什么

小哈设计助理是一个常驻桌面的 AI 助手（桌宠），把「AI 对话 + 钉钉消息 + 语雀文档 + 待办管理 + 素材检索 + **按规范生成/改图**」整合到一个入口，帮设计师在不切换应用的情况下高效处理日常工作。

**核心交互**：桌面上有一只小哈 IP 形象，
- **左键点击**：打开/收起功能面板（底部 dock 切换模块）
- **右键点击**：呼出功能菜单（含 API 接入、偏好设置）
- 面板可拖拽、可缩放

**v2.1 亮点**：
- 🎨 **按品牌规范一键出图（三种板块）**：头图 1170×879、宽幅横幅 702×180、竖版弹窗 594×790，说一句「按蓝莓骑行规范生成一张…」自动识别板块，找合规模板 + AI 重新创作
- 🔀 **两版差异化**：一次出 2 版会从**不同模板**取图，在场景、光线、人物活动感上明显不同，避免两版雷同
- 🖼️ **引用图继续改**：钉钉里**引用**机器人发过的某张结果图说「按这张改…」，能准确取到那张原图迭代
- ⚡ **技能自动加载**：对话里点到某个技能名（如某品牌规范），自动加载并应用；规范 .md 改了即改生图效果
- 💬 **允许闲聊开关**：与工作无关的话题也能像通用 AI 一样自由聊
- 🔎 **语雀全文检索**：按标题/正文跨团队搜索文档
- 🖼️ **生图模型独立配置 + 多模型切换**：生图可用与对话不同的服务和 Key

## 三、快速开始（首次使用三步）

### 第 1 步：启动应用
- 找管理员拿打包好的应用（v1.1.1 universal.dmg，M 芯片和 Intel 通用），或开发环境 `npm start`
- 首次打开若提示"已损坏/无法验证开发者"：把 App 拖进「应用程序」后，终端执行一次 `xattr -cr /Applications/哈啰设计助手.app` 再打开

### 第 2 步：配置 AI 对话模型（必做）
右键小哈 → **API 接入** → **AI 模型**。

**推荐：火山方舟（响应快、稳定）** ⭐
| 字段 | 值 |
|------|------|
| 供应商 | 点「火山方舟」 |
| 地址 | `https://ark.cn-beijing.volces.com/api/v3` |
| API Key | 火山方舟控制台的 API Key（`ark-` 开头） |
| 模型 | 已开通的模型 ID，如 `doubao-seed-2-1-pro-260628`（模型广场/在线推理里复制） |

> ⚠️ 不建议用哈啰内部网关做对话模型——实测响应慢、易超时，会导致意图识别/规范扩写失败。**对话用火山方舟最稳。**

填完点「测试连接」，显示「连接成功」即可。

### 第 3 步：按需接入其他能力
- **生图模型**（要生成/改图必配，见第五章）
- **钉钉机器人**（让机器人处理同事消息，见第六章）
- **素材库 DesignHub**（找素材/按规范改图的后端，用公司账号登录）
- **语雀**（读团队文档，走内置团队 Token）

## 四、AI 助手怎么用

打开 AI 面板，输入框直接提问即可。核心能力：

### 4.1 普通对话 / 需求分析 / 文案 / 文档总结
直接问，或粘贴需求/文档内容让它总结、拆解。

### 4.2 技能自动加载 ⚡（新）
对话里**点到某个技能的名字**（如"按蓝莓骑行规范…"），系统自动匹配技能中心里的对应技能并加载其内容来回答，命中时会提示「🎯 已应用技能「XX」」。
- 前提：该技能已在**技能中心**上传/保存（只在聊天里说过、没存成技能是匹配不到的）。

### 4.3 允许闲聊 💬（新）
偏好设置里有「允许闲聊」开关（默认开）：
- **开**：与哈啰无关的话题（通用知识、生活、编程等）也能像通用 AI 一样自由回答
- **关**：只聚焦设计/工作话题，无关话题礼貌婉拒

### 4.4 生成 / 改图
- **纯文字生成**：说「生成一张 XX 的图/海报」→ 直接出图（需配好生图模型）
- **上传图改图**：拖图进面板 + 说改动 → 图生图
- **按规范生成 banner**：见第五章（推荐用法）

### 4.5 其他
- 左侧「≡」查看/切换历史会话，「+」新建（自动持久化）
- 悬停 AI 回复右下角有复制按钮

## 五、生图模型配置 + 按规范生成 🎨（v2.0 核心）

### 5.1 配置生图模型（与对话模型独立）
API 接入 → **生图模型**区。生图可用与对话**不同的服务和 Key**：

| 字段 | 值（哈啰生图服务示例） |
|------|------|
| 生图模型 | `Doubao-Seedream-4.5` 或 `Doubao-Seedream-5.0-lite`（点标签切换） |
| 生图 Key | 哈啰「AI设计生图助手」服务的 Secret Key（`sk-` 开头） |
| 生图地址 | `https://aibrain-large-model-engine-common.hellobike.cn/v1` |
| 默认尺寸 | 留默认或填 `0`（自适应）；也可填 `1170x879` 等具体尺寸 |

> ⚠️ **Key 和地址必须是同一平台的**，别混用（哈啰 Key 配火山地址会连不上）。
> ⚠️ **GPT-Image 系列只能纯文生图，不支持"图生图/改图"**；要改图/按规范生成请用 **Doubao-Seedream** 系列。

**多模型切换**：生图模型是一排可点标签，点哪个用哪个；输入框输新模型名回车加入；标签上的 **×** 删除。

### 5.2 按品牌规范生成（推荐用法）⭐
**前提**：① 配好 Seedream 生图模型；② DesignHub 已登录；③ 品牌规范已存进技能中心；④ DesignHub 里有该品牌对应板块的合规模板。

**支持三种资源位板块**（说需求时会自动识别，也可点名）：

| 板块 | 尺寸 | 版式 | 触发说法示例 |
|------|------|------|------|
| **头图 / 顶部氛围** | 1170×879 | 左文右图，主视觉铺满全出血 | 「按规范生成一张…的头图/banner」 |
| **宽幅横幅** | 702×180 | 头图延展的宽横幅，左文右图 | 「按规范生成一张…的横幅/宽幅」 |
| **竖版弹窗** | 594×790 | 竖版卡片，主视觉居中偏下，底部 CTA | 「按规范生成一张…的弹窗」 |

**怎么用**：一句话，说清主标题和画面内容，例如：
> 按照蓝莓骑行的规范，帮我输出一张主标题是"带娃出发，一骑长大"的头图，内容是亲子户外骑行

**它会自动**：
1. 从 DesignHub 搜对应板块的**合规模板**（如头图搜"顶部氛围 1170_879"）
2. 以模板作**构图/排版骨架参考**（板块对应的 logo/标题/按钮/主视觉位置与尺寸）
3. **AI 重新创作画面内容和文字样式**（不是在原图上小改），一次出 **2 版**——两版会尽量取**不同模板**，在场景、光线、人物活动感上做差异化
4. 只要一版：说「只要一版 / 出一张」即可，跳过差异版

**上传自己的主视觉**：拖入自行车产品图 + 说板块和文案 → 会先按规范把主视觉摆到对应位置（右侧/居中偏下）再延展生成，避免主体被硬塞到中间。

**继续迭代**：直接说修改要求即可，会基于当前版本改，例如：
> 画面要父与子，两人各骑一辆车，傍晚黄昏感，背向镜头向远处骑，文案全用白色

**引用某张图改**（钉钉）：机器人一次发了 2 版后，**引用**其中一张回复「按这张改，天空再暖一点」，会准确取到被引用那张作原图继续迭代。

### 5.3 关于尺寸
- 说了具体尺寸（如"702×180"）→ 按该尺寸出（会等比生成再精确裁到目标尺寸）
- 没说 → 自动用对应板块规范里的尺寸（头图 1170×879 / 宽幅 702×180 / 弹窗 594×790），不会出方图
- 头图等固定资源位会**忽略消息里的尺寸**，强制按规范规格输出，保证资源位规格一致

### 5.4 效果预期（重要）
- **版式/构图**：由合规模板保证，稳定合规 ✅
- **画面内容/文字艺术样式**：AI 自由发挥，质量不错但**非像素级还原**（logo/安全区等细节以"贴合"为主）
- 想要 100% 像素级合规，是后续「代码精确排版」方向（暂未上线）

## 六、钉钉机器人（实名 AI 助理）

让机器人替你接待同事的钉钉消息，AI 协助解读、拟回复、找素材、按规范生成/改图、转待办。

### 6.1 配置
1. 钉钉开放平台创建**企业内部应用** → 加**机器人**能力 → 消息接收模式选 **Stream 模式**
2. 拿到 AppKey（Client ID）/ AppSecret（Client Secret）
3. 桌宠 → API 接入 → 钉钉机器人，填 AppKey / AppSecret / 机器人名称
4. 点「连接机器人」，日志显示 `connected` 即成功
5. 钉钉后台**发布应用**，并在**「应用可用范围」把要用的同事加进去**（否则同事私聊会提示"权限未配置"）

### 6.2 单聊 vs 群聊
- **单聊**：同事直接私聊机器人
- **群聊**：把机器人**添加进群**（群设置→机器人→添加→选企业内部开发的机器人），群里 **@机器人** 触发
- ✅ 群里发图会正确发在**群内**（不再误发私聊）

### 6.3 每条消息能做什么
进会话后底部按钮：🔍 智能解读 / 💬 拟回复 / 📋 提取任务 / 🤖 接管。

### 6.4 找素材 / 按规范生成 / 改图（钉钉里）
- **找素材**：「找个 520 banner」→ 从 DesignHub 检索回缩略图
- **按规范生成**：「按蓝莓骑行规范生成一张…的头图/宽幅/弹窗」→ 自动识别板块、模板改图出 2 版（同第五章）
- **改图**：发**图 + 指令**（同一条消息最稳）；或对已生成结果继续说修改要求迭代
- **引用图改**：**引用**机器人发过的某张结果图回复修改指令 → 准确锁定那张作原图迭代
- **图文分两条发也能认**：先发图后发文字，会自动回溯最近的图作主视觉

## 十一、常见问题 FAQ

**Q：AI 意图老识别错 / 回复慢 / 超时？**
A：对话模型**换成火山方舟**。哈啰内部网关做对话易超时，会连带影响意图识别和规范扩写。

**Q：生成的图不符合品牌规范（版式/尺寸不对）？**
A：① 用「按规范生成」而非纯文字生成；② 确认品牌规范已存进**技能中心**；③ 确认 DesignHub 已登录且有该品牌合规模板；④ 生图模型用 **Seedream**（GPT-Image 不支持改图）。

**Q：生成的图直接在原图上小改、内容不够新？**
A：已改为"模板只作构图参考、内容 AI 重新创作"。若仍偏保守，多说主体/场景细节，或说「再来一版」换表现。

**Q：一次出的 2 版长得几乎一样？**
A：已优化为两版尽量取**不同模板**做差异化。若某板块 DesignHub 合规模板本身只有一张，则只能出一版（不会静默复用同一张凑数）。多补充场景/主体细节也能拉开差异。

**Q：引用了某张图说"按这张改"，却改错了图？**
A：请用钉钉的**引用回复**功能引用目标图（而非重新描述）；机器人每次发图后会把该图记为当前参考图，引用能准确命中。若还不准，把要改的那张单独转发再跟指令。

**Q：详细修改指令被当闲聊敷衍了？**
A：已修复——有进行中改图任务时，画面描述型指令（如"傍晚/背向/文案白色"）会自动当改图迭代。若仍不识别，把它接在上一次生成之后紧跟着发。

**Q：图片"裂开"显示不出来？**
A：远程图会自动经主进程下载成 base64 再显示；若仍裂开多为网络问题，重试或检查代理。

**Q：钉钉里回复了对方收不到 / 同事私聊提示无权限？**
A：① 机器人需**已发布**；② sessionWebhook 仅 **2 小时有效**，超时让对方再发一条；③ 消息接收模式必须 **Stream**；④ 同事要在应用**可用范围**内。

**Q：生图 Key/地址填了连不上？**
A：检查 **Key 与地址是否同一平台**（哈啰 Key 配哈啰地址、火山 Key 配火山地址，别混）。

**Q：关闭应用弹「A JavaScript error occurred」？**
A：已加全局兜底，新版本关闭不再弹（旧实例关最后一次可能还会）。

## 十二、已知限制（当前版本）

- **像素级版式**：文生图/图生图对规范是"方向性贴合"，非像素级还原（logo/安全区/字号层级细节不保证 100%）。100% 合规需后续「代码精确排版」方向。
- **模板依赖**：按规范生成依赖 DesignHub 里有该品牌的合规模板；无模板则退化为普通生图。
- **网络**：火山方舟/外部 API 偶发 SSL 握手失败，多为代理/网络波动，重试即可。
- **钉钉文件接收**：暂不支持接收 .md/.docx/.pdf 等文件（图片可以）。
- **数据存储**：对话/会话历史明文存本机 `~/.hellobike-pet/`，注意隐私。
- **单点连接**：同一套钉钉凭据同时只应由一个客户端连接。
- **打包更新**：Mac 未签名，暂无自动更新，新版本需手动覆盖安装。
